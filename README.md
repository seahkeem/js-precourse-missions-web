# js-precourse-missions-web

프리코스 과제 3개(로또 발매기, 자동차 경주, 문자열 덧셈기)를 웹 기반 애플리케이션으로 통합 구현한다.

## 1. 목표
- Vanilla JavaScript 핵심 로직(Domain Logic)을 웹 환경에 재사용할 수 있도록 구조화한다.
- 객체 지향 설계(OOP) 원칙을 철저히 적용하여 세 개의 독립된 기능을 모듈화하고 통합한다.
- 사용자 친화적인 웹 인터페이스(HTML/CSS) 및 비동기 이벤트 처리를 구현한다.
- 테스트 주도 개발(TDD) 방법론을 준수하여 단위 테스트를 작성한다.

## 2. 구현 과제 목록 및 추가 제한
기존 프리코스 요구사항 외에 웹 환경 및 도메인을 반영한 새로운 제한을 설정했다.

| 과제 | 제한/도전 목표 | 이유 및 근거 |
|------|----------------|--------------|
| 문자열 덧셈기 | 입력 문자열 길이를 최대 12자로 제한 | 계산기 UI에서 한 번에 입력하기 쉬운 적절한 문자열 길이 |
| 자동차 경주 | 경주 자동차 수를 최대 10대로 제한 | UI 표시 한계 |
| 자동차 경주 | 총 시도 횟수를 최대 22회로 제한 | F1 그랑프리 시즌 횟수(22회 내외) 차용 |
| 로또 발매기 | 최대 구입 개수 5회 (5,000원)로 제한 | 온라인 로또 구매 제한 기준(최대 5천 원) 반영 |

## 3. 기술 스택
- 언어: JavaScript, HTML, CSS
- 개발환경: Node.js
- 번들러: Vite
- 테스트:Jest


## 4. 실행 방법
```
git clone https://github.com/seahkeem/js-precourse-missions-web.git
cd js-precourse-missions-web

npm install 

npm run start
```
